===============================================================================
                        커플 FootPrint 앱 - 요구사항 명세서
===============================================================================

프로젝트명: 커플 FootPrint (Couple Restaurant Map 확장)
작성일: 2025-09-09
목적: 애인과 함께 방문한 모든 장소를 기록하고 공유하는 추억 아카이브 앱

===============================================================================
                                1. 개요
===============================================================================

1.1 프로젝트 배경
- 기존 couple-restaurant-map 프로젝트를 확장하여 맛집뿐만 아니라 모든 방문 장소를 기록
- FootPrint 스타일의 위치 기록 앱으로 발전
- 커플이 함께 다녀온 모든 추억을 지도와 타임라인으로 관리

1.2 주요 개념
- 사후 기록 방식: 집에서 여유롭게 기록
- 위치 + 사진 + 추억: 종합적인 기록 시스템
- 지도 뷰 + 타임라인 뷰: 이중 인터페이스 제공

===============================================================================
                            2. 사용자 요구사항
===============================================================================

2.1 핵심 사용자 시나리오
- 데이트 후 집에 와서 여유롭게 오늘 다녀온 장소들을 기록
- 맛집, 카페, 공원, 영화관, 쇼핑몰 등 모든 방문 장소 기록 가능
- 사진과 함께 추억을 저장하여 나중에 회상 가능
- 애인과 함께 타임라인을 보며 지난 추억들을 되돌아보기

2.2 사용자 페르소나
- 주 사용자: 20-30대 연인 커플
- 사용 상황: 데이트 후 기록, 추억 회상, 다음 데이트 장소 계획
- 기술 수준: 스마트폰 사용에 능숙한 일반 사용자

===============================================================================
                            3. 기능 요구사항
===============================================================================

3.1 기존 기능 유지 (couple-restaurant-map)
✓ 구글 지도 기반 위치 표시
✓ 커플 코드 시스템으로 데이터 공유
✓ localStorage → Supabase 클라우드 마이그레이션
✓ 맛집 등록, 평점, 메모 기능
✓ 통계 및 추천 시스템

3.2 새로운 핵심 기능

3.2.1 위치 카테고리 시스템
- 맛집(🍽️): 기존 레스토랑 기능 유지
- 카페(☕): 커피숍, 디저트카페 등
- 공원(🌳): 산책로, 놀이공원, 테마파크 등  
- 영화관(🎬): 영화관, 공연장, 전시관 등
- 쇼핑(🛍️): 쇼핑몰, 시장, 매장 등
- 문화시설(🎨): 박물관, 미술관, 도서관 등
- 기타(📍): 분류되지 않는 모든 장소

3.2.2 다중 사진 업로드
- 장소당 여러 장의 사진 첨부 가능
- 음식 사진, 분위기 사진, 셀카 등
- 이미지 압축 및 최적화
- 사진 갤러리 뷰어

3.2.3 타임라인 뷰
- 시간순으로 방문 기록 표시
- 날짜별 그룹핑
- 사진 썸네일과 장소명, 간단 메모 표시
- 월별, 연도별 네비게이션
- 특별한 날 하이라이트 기능

3.2.4 방문 날짜 관리
- 실제 방문 날짜 기록
- 달력 인터페이스로 날짜 선택
- 날짜별 필터링 및 검색

===============================================================================
                            4. 기술 요구사항
===============================================================================

4.1 시스템 아키텍처
- 프론트엔드: HTML5, CSS3, JavaScript (Vanilla)
- 백엔드: Supabase (Database + Auth + Storage)
- 지도 API: Google Maps JavaScript API
- 호스팅: Vercel 또는 Netlify

4.2 데이터베이스 스키마 확장
기존 restaurants 테이블 → locations 테이블로 변경

필드 추가:
- category: TEXT (위치 카테고리)
- photos: TEXT[] (사진 URL 배열)  
- visit_date: DATE (실제 방문 날짜)

4.3 성능 요구사항
- 지도 로딩: 3초 이내
- 사진 업로드: 10MB 이하, 압축 처리
- 타임라인 무한 스크롤 지원
- 모바일 반응형 디자인

===============================================================================
                            5. 사용자 인터페이스 요구사항
===============================================================================

5.1 네비게이션 구조
- 지도 뷰 / 타임라인 뷰 탭 전환
- 카테고리별 필터 버튼
- 검색 기능 (장소명, 날짜)

5.2 지도 뷰 개선
- 카테고리별 마커 아이콘 차별화
- 마커 클러스터링 (위치가 많을 때)
- 카테고리 필터 토글

5.3 위치 추가 폼 개선
- 카테고리 선택 드롭다운
- 다중 사진 업로드 인터페이스
- 방문 날짜 선택 달력
- 메모 및 평점 입력

5.4 타임라인 뷰 (신규)
- 세로 타임라인 레이아웃
- 날짜 헤더 + 위치 카드
- 사진 갤러리 모달
- 월/년 단위 네비게이션
